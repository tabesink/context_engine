Design guidelines recreated from the reference images

Use these as direct instructions for the coding agent when building Context Engine surfaces.

control radius 8px
01

Canvas-first layout

Use large white work areas with no heavy containers. Let task lists, code panes, and workflow columns float in whitespace.

#fff #fafafa
02

Medium radius, not pill

Buttons, fields, chips, rows, tabs, and toolbar segments should use 8px radius. Cards can use 12px.

controls 8px panels 12px
03

Compact task typography

Use 13–14px task labels, 10–12px mono metadata, and tight vertical rhythm. Avoid large form chrome.

13px UI mono metadata
04

Status as small signals

Use tiny green, blue, orange, red dots for done, running, paused, and risk. Never use full-width alert color blocks.

done run pause

Component grammar

Prefer plain panels, task tables, step rails, workflow columns, code panes, and floating toolbars. Keep borders to 1px and shadows off except for the bottom tool palette.

What changed from the earlier preview

This version moves away from the Ollama-style full pill language. The new look follows the attached agent-workbench references: square enough to feel technical, still soft enough to feel modern.
